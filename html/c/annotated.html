tmon C API  0.1
Data Structures
Here are the data structures with brief descriptions:
 Cezstr_sSimple, "easy" string object, used internally by tmon. Not intended as a supported API
 Ctmon_conn_sTmon connection monitoring object. Created by tmon_conn_create(). Create one per receiver delivery controller from inside the source_notification_function (receiver) callback. That callback can return this object as the source-specific clientd. See Client Data Pointers. Application must not write to fields in this structure
 Ctmon_ctx_sTmon context monitoring object. Created by tmon_ctx_create(). Create one of these per application context immediately after creating each context. Application must not write to fields in this structure
 Ctmon_rcv_sTmon receiver or wildcard receiver monitoring object. Created by tmon_rcv_create(). Create one per application receiver immediately before creating each receiver. Application must not write to fields in this structure
 Ctmon_src_sTmon source monitoring object. Created by tmon_src_create(). Create one per application source immediately before creating each source. Application must not write to fields in this structure
