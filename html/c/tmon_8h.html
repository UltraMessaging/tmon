◆ tmon_conn_create()

tmon_conn_t* tmon_conn_create ( tmon_rcv_t *  tmon_rcv,
const char *  source_str 
)

Create a tmon connection monitoring object to monitor an application UM receiver's delivery controller. A delivery controller can be thought of as "connection" between a source to a topic and a receiver of that same topic. This should be called inside the source_notification_function (receiver) "create" callback. That callback can return this object as the per-source client data. See Client Data Pointers.

Parameters
tmon_rcvtmon receiver monitoring object. This must be the tmon object associated with the same application context as the application source being created.
source_strsource string for this connection, including Topic Index.
Returns
tmon connection monitoring object. This should be returned as the per-source client data

◆ tmon_create_context()

lbm_context_t* tmon_create_context ( char *  topic_str,
char *  config_file,
int  buf_lens,
char *  transport_opts 
)

Create a UM context based on a standard UM "transport options" string. The configuration file and topic string are extracted from the transport options and returned to the caller, along with the UM context. This information is subsequently passed to tmon_create_monrcv(). Delete this context with the standard UM lbm_context_delete() API.

Parameters
topic_strPointer to a byte buffer to be filled with the topic string taken from the transport options.
config_filePointer to a byte buffer to be filled with the name of the configuration file taken from the transport options.
buf_lensSize of the topic_str and config_file buffers.
transport_optsUser specified transport options which are passed to the standard UM monitoring software.
Returns
UM context.

const char* tmon_ctime ( char *  buffer,

time_t  cur_time 
)

Portable, reentrant converter from Posix "time_t" to ascii date/time.

"time_t" is number of seconds since the Unix Epoch (midnight, 1-Jan-1970).

Parameters
bufferByte array at least 27 bytes long. Date/time string will be written here.
bufsizeActual number of bytes in "buffer".
cur_timePosix time value to convert.
